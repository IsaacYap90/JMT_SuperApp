Isaac's ask (msg 5353 → 5359): hide part-time coaches from the leave tab entirely, keep Isaac as the only full-time employee with leave records, let Jeremy (admin) add/cancel MC on Isaac's profile without needing a new role.

- `src/app/actions/leave.ts`
  - `submitLeave` now accepts optional `target_coach_id`. If set and caller is admin/master_admin, the leave is created on the target coach's behalf (caller role checked via `users.role`); otherwise 403

  - `cancelLeave` now bypasses ownership + past-date guards when the caller is admin/master_admin

- `src/components/leave-page-client.tsx`
  - `targetCoachId` state added
  - `canCancel` returns `true` for admins (was `false`)
  - New `openAdminLeaveForm(coachId)` seeds `targetCoachId` + defaults `leaveType` to `sick`
  - `handleSubmit` passes `target_coach_id: targetCoachId || undefined` to `submitLeave`
  - New blue "+ Apply Leave" button on each coach card (admin-only)
  - Form visibility gate: `{showForm && (!admin || targetCoachId) && ...}` — admin form only opens when they clicked a specific coach's card
  - Form header shows "Applying leave for {coach name}" in admin target mode

- **Part-time coaches hidden from leave entirely, visible in schedule tab only** — they don't accrue leave; schedule page still uses them for class assignments. Filter applied at DB level (inner join), not client-side.

- **Admin form only opens per-coach card** — prevents ambiguity about whose leave is being applied. Clicking the top-level "+ Apply Leave" button wouldn't know which coach.
