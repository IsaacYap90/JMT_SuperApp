## What broke (and how it was fixed)

- **Build failure on `Set` spread iteration** in `adminCreateTrialBooking`. TS target too low for `[...set]`. Fixed by switching to `forEach` + array push.
- **`Telegram 400 Bad Request: can't parse entities — '(' must be escaped`**. Cron route's per-user message contained `(lead)` and `(assistant)` which the existing markdown helper *should* have escaped, but in practice the daily-schedule message was rejected and the helper logged the error then returned silently while the route's `sent` counter still ticked up. Misleading green return, no Telegram delivery. Fixed by introducing `sendTelegramPlainToUser` (no `parse_mode`), routing the daily schedule through it, and having the cron route increment `sent`/`skipped` based on the boolean return.
- **Local preview couldn't deliver Telegram messages** because `.env.local` doesn't contain `JMT_TELEGRAM_BOT_TOKEN` or `JMT_TELEGRAM_USER_MAP` (CLAUDE.md says don't modify `.env.local`). Pulled values from `vercel env pull` to a `/tmp` file, exported them inline when restarting `next start`, deleted the temp file. Worked.
- **Off-by-one wording**: Isaac said "remove my role" — first instinct was to remove role labels for everyone always-assistant. Hardcoded his UUID in `HIDE_ROLE_USER_IDS` instead, so other coaches still see role labels (Jeremy's vary).
- **Wrong log capture**: backgrounded `next start` lost stderr → cron failures invisible until I tested the bot token + chat_id directly with `curl`, then ran the escape regex in `node -e` to confirm parens were the issue.

## Decisions made

- **Two-step confirm for destructive actions** (PT client delete) — pattern to reuse for any future delete in this dashboard
- **Idempotency on review/approve actions** — DB-level via status filter on the `update` query, not just an early return. Both layers, so a stale UI state can never re-trigger notifications
- **Plain text Telegram for dynamic messages** — anything with user-supplied content (class names, member names, free-form text) goes through `sendTelegramPlainToUser`. MarkdownV2 helper stays for fixed-format alerts (PT scheduled, leave decisions)
- **Cron auth via `CRON_SECRET` Bearer token** — Vercel Cron only. Local fires bypass auth if env var is unset, and use `?only=<userId>` for targeted test sends
- **Bundle, don't deploy** — held the entire 7-item bundle on a Cloudflare Tunnel link for Isaac to test on his phone before any prod push
